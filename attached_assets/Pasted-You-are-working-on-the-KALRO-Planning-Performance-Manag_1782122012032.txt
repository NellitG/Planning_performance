

# RELATIONSHIP REQUIREMENTS

Implement the following hierarchy:

```text
Key Result Area (KRA)
        ↓
Strategic Objective
        ↓
Strategy
        ↓
Key Activities
        ↓
Expected Outputs
        ↓
Output Indicators
        ↓
Projects
```

Rules:

* A Strategic Objective must belong to a Key Result Area.
* A Strategy must belong to a Strategic Objective.
* A Key Activity must belong to a Strategy.
* An Expected Output must belong to a Strategy.
* An Output Indicator must belong to a Strategy.
* Projects can be linked to one or more KRAs, Strategic Objectives, and Strategies.

---

# DYNAMIC DATA BEHAVIOR

All dropdowns and selections must be populated from previously created records.

Examples:

* Expected Outputs should be linked to a Strategy.
* Output Indicators should be linked to a Strategy.
