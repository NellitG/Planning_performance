# PROJECT DETAILS WORKFLOW

Implement a multi-step workflow.

## STEP 1 – PROJECT DOCUMENT UPLOAD

When a project is opened, display Step 1.

### Interface Requirements

Show:

* Project information summary
* File Upload Component

The upload component should support:

* Drag and drop upload
* File selection button
* Upload progress indicator
* Uploaded file preview

Supported documents may include:

* Project proposal
* Concept note
* Supporting documents
* Approval documents

  Note: The document uploaded should can be viewed below

---

### Validation Rule

Users must upload at least one project document before proceeding.

Disable the **Next Step** button until:

```text
Project Document Uploaded = TRUE
```

If no file is uploaded:

* Show validation message
* Prevent navigation to Step 2

---

### Navigation

Display:

* Next Step Button

The button becomes active only after a successful upload.

---

## STEP 2 – COMPONENT, STRATEGIC OBJECTIVE & STRATEGY MAPPING

When Step 1 is completed, allow navigation to Step 2.

---

### Component Selection

Display a dropdown:

```text
Select Component
▼ Component A
▼ Component B
▼ Component C
```

Load all Components previously created in the Components module.

---

### Strategic Objectives Table

When a Component is selected:

Automatically load all Strategic Objectives associated with that Component.

Display them in a table.

Suggested columns:

| Strategic Objective | Date Created |
| ------------------- | ------------ |

Example:

```text
Component: Agricultural Productivity

Strategic Objectives

1. Increase Crop Productivity
2. Improve Seed Quality
3. Enhance Farmer Adoption
```

---

### Strategy Selection

Below the Strategic Objectives table:

Display all Strategies associated with the selected Strategic Objectives.

Allow users to select one or more strategies.

Example:

```text
☐ Strategy A
☐ Strategy B
☐ Strategy C
```

or

```text
Select Strategy
▼ Strategy A
▼ Strategy B
▼ Strategy C
```

---

### Save Mapping

When a strategy is selected:

Create and save the relationship:

```text
Project
    ↓
Component
    ↓
Strategic Objective
    ↓
Strategy
```

The selected project should now be linked to:

* Component
* Strategic Objective
* Strategy

---

## DATA RELATIONSHIPS

Ensure the following hierarchy is maintained:

```text
Component
    ↓
Strategic Objectives
    ↓
Strategies
    ↓
Projects
```

Requirements:

* Components populate Strategic Objectives.
* Strategic Objectives populate Strategies.
* Strategies can be linked to Projects.
* Relationships must persist in the database.
