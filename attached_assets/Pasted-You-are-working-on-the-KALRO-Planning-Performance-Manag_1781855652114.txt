Current Issue:

The Strategy module correctly allows multiple strategies to be created.

However:

If a user creates multiple strategy records, the dashboard/listing incorrectly displays duplicate totals or counts each entry separately when it should be grouped appropriately.

Example:

```text
Component A
  Strategic Objective A
     Strategy 1
     Strategy 2
```

The listing currently shows:

```text
Total Strategies: 2
```

Required Behavior:

Display the parent strategy grouping as a single record while still allowing multiple child strategies underneath it.

Example:

```text
Component A
    Strategic Objective A
        Strategy 1
        Strategy 2
```

Dashboard should display:

```text
Total Strategy Groups: 1
```

while allowing users to expand or view all associated strategies.

Review:

* strategy count calculations
* dashboard statistics
* listing logic
* grouping queries

Ensure duplicate counting is eliminated.
